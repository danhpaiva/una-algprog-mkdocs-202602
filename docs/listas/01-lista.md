# 🚀 Lista 01 — Sua Identidade Digital e Primeiro Repositório

Agora que você entendeu que programar é **decompor problemas** e criar **algoritmos**, chegou a hora de preparar o seu kit de ferramentas de desenvolvedor.

Nesta tarefa, vamos utilizar o **GitHub**, que funciona como o "LinkedIn dos programadores" e será o nosso principal portfólio de código durante o semestre.

---

## 🛠️ O Algoritmo da Entrega

### 1. Criar sua Base de Operações (GitHub)

Se ainda não tem, crie uma conta no [GitHub](https://github.com/).

* **Dica:** Use um nome de usuário profissional — você o levará para a carreira.

### 2. Networking de Código

Procure e siga seus colegas de turma no GitHub. Um bom desenvolvedor nunca está sozinho.

### 3. Criar o Repositório da Disciplina

Crie um repositório **público** com o nome exato:
`una-algprog-lista01`

> **Por que esse nome?** A padronização facilita a organização e a correção pelo professor.

### 4. Subir seu Algoritmo

Lembra dos desafios do **"Mestre Cuca"** ou da **"Travessia do Rio"** da [Aula 01](../aulas/01-aula.md)?

* Crie um arquivo (`.txt` ou `.md`) com a solução do algoritmo que você desenvolveu.
* Faça o *commit* desse arquivo para o seu novo repositório.

### 5. Entrega Final no Google Classroom

1. Copie a **URL do repositório** (ex: `https://github.com/seu-usuario/una-algprog-lista01`).
2. Vá até a tarefa correspondente no **Google Classroom**.
3. Cole o link e marque como entregue.

---

## ⚠️ Checklist de Sucesso

!!! check "Verifique antes de enviar"
    - [ ] O repositório está **público**? (Se for privado, o professor não consegue ver!)
    - [ ] O nome do repositório está correto?
    - [ ] O arquivo com o algoritmo está legível dentro do GitHub?

---

!!! info "Dúvidas?"
    Se o "algoritmo de criação de conta" der erro, peça ajuda da turma ou chame o professor!
